# Contributing

## Development

Build, test, run, and release documentation now lives in [`DEVELOP.md`](./DEVELOP.md).

## Pull Requests

- Keep changes focused and easy to review.
- Include tests when touching git parsing, worktree handling, persistence, or layout logic.
- Call out any manual verification you performed for sidebar, pane, or terminal behavior.
- Avoid changing vendored binaries in `Argo/Vendor/` unless the change explicitly requires it.
- Keep `docs/` in sync when changing terminal layering, testing conventions, or contributor workflows.

## Discussions

- Bug reports and feature requests: <https://github.com/krystal/argo/issues>
- Security reports: see `SECURITY.md`
