# Twilight Terminal

一套**单色驱动**的透明终端主题系统。给一个颜色，用色彩理论现场派生出整套主题——强调色、终端 16 色、日落壁纸全部联动。灵感来自「日落天鹅湖」壁纸：暖色落日光 + 冷色水面。

> 核心特性：**选任意一个颜色 → 生成完整协调的终端主题 → 一键导出成 Warp 能用的文件。**

## 快速上手

### 想直接用预设（最快）

```bash
mkdir -p ~/.warp/themes
cp warp-themes/*.yaml ~/.warp/themes/
```
然后 Warp 里 `Settings → Appearance → Themes` 选主题；`→ Window` 设透明度 80–88% + 开模糊。

### 想用自己的颜色

```bash
open preview.html
```
右下角控制条：点彩虹环取色器 / 直接输 hex → 整个界面实时重算 → 满意后点「导出 Warp」→ 下载的 `.yaml` 丢进 `~/.warp/themes/`。

### 想让 AI 重建这个界面

把 `设计规范.md` 喂给 AI。里面有完整算法代码、所有尺寸色值、组件状态和验收清单，AI 读完能 1:1 实现，不需要再做设计决策。

## 三个设计要点

**1. 单色驱动。** 整个视觉系统由一个种子色生成。次要色按色相环固定角度偏移（互补 +168°、类比 ±96°），保证任何种子出来的配色都和谐。亮度有护栏，再暗的种子色落到暗背景也可读。

**2. 透明只让一层做。** 终端软件的窗口透明 + 模糊是系统级的，原生不卡。主题文件**只管颜色**，不碰透明/模糊。两者分开，避免双重模糊、颜色发灰。这是 `preview.html` 和真实 Warp 的共同纪律。

**3. 终端语义色不失真。** `red/green/blue` 在终端里有强语义（`git diff`、`ls`、报错靠它）。算法把这些色**锚定标准色相**、只被种子色轻染 12%——保证任何主题下「红是红、绿是绿」。这是可用性底线，不为视觉统一牺牲。

## 说明

- `preview.html` 里的壁纸是 CSS 渐变模拟的，用于预览不同种子色的氛围。真实使用时壁纸是你自己的桌面图，主题只换终端配色。
- 预览器导出的 YAML 与界面预览**同源同算法**，预览什么色导出就是什么色。
- 主题 YAML 只能定义终端正文配色；Warp 自带的侧栏、命令面板等 chrome 不开放给主题改。`preview.html` 里重排的那套布局属于「理想设计」，真实 Warp 落不了地——但配色完全可以落地。
